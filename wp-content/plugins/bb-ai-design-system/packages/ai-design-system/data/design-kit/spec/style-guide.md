## Style Guide Content Structure

A style guide page showcases a design system's visual vocabulary. Showcase the design's vocabulary in distinct sections (palette, typography, components, etc.) so individual parts can be iterated on independently. Pick what suits the design.

### Using Variations in Showcase Sections

Many showcase sections need items that look visually different from each other (different swatch colors, different bar widths, different button styles). Use a descriptive class on each item — see the Item Variations section in the annotation spec. Never use inline styles (`style="background: ..."`) for per-item visual differences; the annotation system does not support dynamic style attributes.
