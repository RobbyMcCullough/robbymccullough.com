# Bundled Design Kits

Runtime-imported kits served by `BuiltInKitRegistry`. Each subdirectory listed in `_manifest.json` is exposed through the REST API and imported into the user's WordPress site via the standard kit pipeline.

## What goes in a bundled kit

Only files the server actually reads:

- `kit.json` — identity manifest (`uuid`, `name`, `description`)
- `design-system/` — `styles.css`, optional `script.js`, optional `art-direction.md`
- `globals/` — `header.html`, `footer.html` (optional)
- `pages/` — one HTML file per page (`index.html` is the homepage)

Kit-specific design notes (e.g. `RATIONALE.md`) are fine to include — they're ignored at runtime and useful for whoever maintains the kit.

## What does NOT go in a bundled kit

- **`spec/`** — the format contract and art direction guidance. Reference the canonical copy at [`../design-kit/spec/`](../design-kit/spec/) instead. Keeping per-kit copies causes drift: when the spec evolves, the shipped kits fall out of date silently.
- **`AGENTS.md`, `CLAUDE.md`** — same reason. The canonical copies live at [`../design-kit/AGENTS.md`](../design-kit/AGENTS.md) and [`../design-kit/CLAUDE.md`](../design-kit/CLAUDE.md).

If an agent needs to edit a bundled kit in this repo, point it at `../design-kit/` for format rules.

## Adding a new bundled kit

1. Copy `../design-kit/` as your starting point.
2. Delete `spec/`, `AGENTS.md`, and `CLAUDE.md` from the copy — those belong only in the download template.
3. Set a unique `uuid` in `kit.json` (`crypto.randomUUID()`).
4. Author pages, globals, and design system as normal.
5. Add a manifest entry to `_manifest.json`:

```json
{
  "id": "your-kit-slug",
  "name": "Your Kit",
  "description": "Short description."
}
```

The `id` must match the directory name and be lowercase kebab-case.

## Keeping the format contract fresh

The canonical spec is generated by `scripts/build-design-kit.mjs` from source files in `../spec/core/`. If you change any of those sources, re-run the build script so `../design-kit/spec/format-contract.md` stays current. CI will fail if the generated file is stale.
