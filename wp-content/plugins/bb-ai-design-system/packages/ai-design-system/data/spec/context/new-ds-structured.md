### Creating a New Design System

No site design system exists yet. Set `mode: "create_ds"` and fill `design_system` in the tool call. The tokens, reset, base CSS, base JS, and fonts you produce become the site-wide design system every future page on this DS will inherit.

Required `design_system` fields:

- `label` ‚Äî short title for the design system (e.g., "Acme Marketing", "Coastal Bakery").
- `tokens` ‚Äî flat map of `--ds-{category}-{name}` keys to raw CSS literal values. Required categories: `color`, `font`, `space`, `text`, `weight`. Optional: `radius`, `shadow`, `transition`, `line`, `letter`, `width`. One flat object ‚Äî no `[data-X]` selectors, no media queries, no `:where()`, no alternate-theme blocks. The storage layer drops anything that isn't a flat key/value pair.
- `reset` ‚Äî reset CSS that runs before base. Start from a standard reset (box-sizing, margin zeroing, heading inheritance, link inheritance, fluid `<img>`, blockquote reset). Add element resets only if the design needs them.
- `base_css` ‚Äî reusable classes future pages on this DS will inherit by name. Use the `bb-` prefix on reusable classes (`.bb-container`, `.bb-btn`, `.bb-badge`). Keep base small ‚Äî most styling belongs in section CSS. **If a class will be used across multiple sections, define it here** so future pages get it for free, not in `page.css`.
- `base_js` ‚Äî DS-durable JS: an `onReady` helper, breakpoint helpers, and any cross-cutting wiring keyed to generic classes you defined in base CSS (for example, an IntersectionObserver over a `.bb-reveal` class ‚Äî self-invoke kickoff so future pages opt in by adding the class). Pair observers with a `MutationObserver` over `document.body` since page DOM is dynamic.
- `fonts` ‚Äî array of `{ family, variants }` objects. Family names as they appear in Google Fonts (e.g., `"Inter"`, `"Playfair Display"`). `variants` is the opaque segment after the colon in a Google Fonts URL (e.g., `"ital,wght@0,400;0,700;1,400"`). Empty string is fine ‚Äî the importer falls back to a sensible weight set. Request only weights actually used in your CSS, and cover every `--ds-weight-*` token.

**`base_css`/`base_js` is per-DS; `page.css`/`page.js` is per-page.** Anything in `design_system.base_css` or `design_system.base_js` becomes part of the design system and runs on every future page. **The same logic applies to any class your design uses across multiple sections:** define the rule once in `base_css` so every future page inherits it without redefining. Reach for `page.css` / `page.js` only when the behavior is genuinely specific to this one page and won't generalize ‚Äî a one-off countdown, an image lightbox initialized only on a gallery page, a page-scoped utility class, or an observer keyed to selectors no other page would use. When in doubt, prefer `base_css` / `base_js`: future pages get it for free, and there's no duplication if another page wants the same effect.

**Whatever shape of spec the user provides** ‚Äî YAML frontmatter (Google DESIGN.md), free-form markdown with embedded CSS, JSON token files (DTCG / Style Dictionary), plain CSS dumps, or hybrid ‚Äî interpret its intent and emit our structured shape. The work of mapping author conventions to our token namespace happens here, not at runtime. Apply the canonicalization rules in `core/design-tokens.md` so the structured `tokens` map uses `--ds-{category}-{name}` regardless of what the source spec called things.

**Variants.** If the source spec defines multiple values for the same role via runtime selectors (`[data-brand="X"]`, `[data-theme="dark"]`), media queries (`prefers-color-scheme`), or sibling theme files, treat these as alternative brand seeds rather than within-DS variants. A WordPress install is one site; one brand applies. Pick the canonical brand ‚Äî asking the user which one if ambiguous, defaulting to the parent or first-listed brand if obvious from context ‚Äî set those values as the `tokens` defaults. Do not emit `[data-X]` selectors or media-query token blocks anywhere; the storage layer does not preserve them and will surface a warning if they appear.
